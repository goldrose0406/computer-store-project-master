📝 1. AUTHENTICATION ENDPOINTS
===========================

✅ TEST 1.1: Register New User (Người dùng mới)
  ─────────────────────────────────────────────

  METHOD: POST
  URL: http://localhost:5000/api/auth/register
  
  INPUT (Body - JSON):
  {
    "name": "Nguyễn Văn A",
    "email": "user@example.com",
    "password": "123456",
    "confirmPassword": "123456"
  }
  
  EXPECTED OUTPUT (Status: 201):
  {
    "message": "User registered successfully",
    "userId": 1,
    "email": "user@example.com"
  }
  
  ❌ LỖI CÓ THỂ GẶP:
  - Status 400: Email already exists
  - Status 400: Password not match
  - Status 400: Missing required fields


✅ TEST 1.2: Login (Đăng nhập)
  ────────────────────────────

  METHOD: POST
  URL: http://localhost:5000/api/auth/login
  
  INPUT (Body - JSON):
  {
    "email": "admin@computerstore.com",
    "password": "Admin@123"
  }
  
  EXPECTED OUTPUT (Status: 200):
  {
    "message": "Login successful",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "email": "admin@computerstore.com",
      "role": "admin"
    }
  }
  
  💾 GHI NHỚ TOKEN: Lưu token này, dùng cho các request cần authorization
  
  ❌ LỖI CÓ THỂ GẶP:
  - Status 401: Invalid email or password
  - Status 400: Missing email/password


✅ TEST 1.3: Verify Token (Xác thực token)
  ─────────────────────────────────────────

  METHOD: GET
  URL: http://localhost:5000/api/auth/verify
  
  HEADER:
    Authorization: Bearer {JWT_TOKEN_TỪ_LOGIN}
  
  EXPECTED OUTPUT (Status: 200):
  {
    "message": "Token valid",
    "user": {
      "id": 1,
      "email": "admin@computerstore.com",
      "role": "admin"
    }
  }
  
  ❌ LỖI CÓ THỂ GẶP:
  - Status 401: Token not provided
  - Status 401: Invalid or expired token

📦 2. PRODUCTS ENDPOINTS
===========================

✅ TEST 2.1: Get All Products (Danh sách sản phẩm)
  ────────────────────────────────────────────────

  METHOD: GET
  URL: http://localhost:5000/api/products
  
  OPTIONAL QUERY PARAMETERS:
    - brand: "Laptop" hoặc "Desktop" (filter)
    - sortBy: "price", "rating", "newest"
    - page: 1 (default)
    - pageSize: 12 (default)
  
  EXAMPLE với filter:
    http://localhost:5000/api/products?brand=Laptop&sortBy=price&page=1&pageSize=10
  
  EXPECTED OUTPUT (Status: 200):
  {
    "products": [
      {
        "id": 1,
        "name": "Laptop Dell XPS 13",
        "brand": "Laptop",
        "price": 999.99,
        "rating": 4.5,
        "description": "High performance laptop",
        "image": "url-to-image"
      },
      {
        "id": 2,
        "name": "Desktop HP Pavilion",
        "brand": "Desktop",
        "price": 799.99,
        "rating": 4.2,
        ...
      }
    ],
    "totalProducts": 50,
    "page": 1,
    "pageSize": 12,
    "totalPages": 5
  }
  
  ✅ TEST CÁC QUERY PARAMETERS:
    1. Không tham số: http://localhost:5000/api/products
    2. Filter brand: http://localhost:5000/api/products?brand=Laptop
    3. Sort by price: http://localhost:5000/api/products?sortBy=price
    4. Pagination: http://localhost:5000/api/products?page=2&pageSize=10


✅ TEST 2.2: Get Product By ID (Chi tiết sản phẩm)
  ────────────────────────────────────────────────

  METHOD: GET
  URL: http://localhost:5000/api/products/1
  
  INPUT: ID = 1 (thay thế trong URL)
  
  EXPECTED OUTPUT (Status: 200):
  {
    "id": 1,
    "name": "Laptop Dell XPS 13",
    "brand": "Laptop",
    "price": 999.99,
    "rating": 4.5,
    "description": "High performance laptop",
    "image": "url-to-image",
    "createdAt": "2026-03-28T10:00:00Z"
  }
  
  ❌ LỖI CÓ THỂ GẶP:
  - Status 404: Product not found (ID không tồn tại)
  - Status 400: Invalid product ID


✅ TEST 2.3: Create Product (Thêm sản phẩm - ADMIN ONLY)
  ──────────────────────────────────────────────────────

  METHOD: POST
  URL: http://localhost:5000/api/products
  
  HEADER:
    Authorization: Bearer {ADMIN_TOKEN}
    Content-Type: application/json

    "name": "Laptop Asus ROG Gaming",
    "brand": "Laptop",
    "price": 1599.99,
    "rating": 4.8,
    "description": "High-end gaming laptop with RTX 4090",
    "image": "https://example.com/image.jpg"
  }
  
  EXPECTED OUTPUT (Status: 201):
  {
    "message": "Product created successfully",
    "product": {
      "id": 51,
      "name": "Laptop Asus ROG Gaming",
      "brand": "Laptop",
      "price": 1599.99,
      "rating": 4.8,
      "description": "High-end gaming laptop with RTX 4090",
      "image": "https://example.com/image.jpg",
      "createdAt": "2026-03-28T10:30:00Z"
    }
  }
  
  ❌ LỖI CÓ THỂ GẶP:
  - Status 401: Token not provided (chưa login)
  - Status 403: Not authorized (role không phải admin)
  - Status 400: Missing required fields (name, price, brand)

  URL: http://localhost:5000/api/orders/1

  {
    "status": "confirmed"
  }

    - "delivered" (đã giao)

  {
    "message": "Order updated successfully",
    "order": {
      "id": 1,
      "userId": 1,
      "productId": 5,
      "quantity": 2,
      "status": "confirmed",
      "totalPrice": 1999.98,
      "updatedAt": "2026-03-28T11:00:00Z"
    }
  }
  
  ❌ LỖI CÓ THỂ GẶP:
  - Status 403: Not authorized (không phải admin)
  - Status 404: Order not found
  - Status 400: Invalid status value

PHẦN 7: TROUBLESHOOTING
================================================================================

❓ Vấn đề 1: 404 Not Found khi truy cập /api-docs
  Solution: Kiểm tra Backend đã khởi động: npm run dev

❓ Vấn đề 2: Connection Refused (5000 port)
  Solution: 
  - Port 5000 bị chiếm: lsof -i :5000 (Mac/Linux) hoặc netstat -an | findstr :5000 (Windows)
  - Hoặc đổi port trong .env file

❓ Vấn đề 3: Token không hợp lệ
  Solution: 
  - Token hết hạn (timeout)
  - Sao chép token không đúng
  - Không thêm "Bearer " prefix

❓ Vấn đề 4: CORS Error
  Solution:
  - Đảm bảo Backend đã enable CORS
  - Kiểm tra server.js: app.use(cors())

❓ Vấn đề 5: Database Error
  Solution:
  - Kiểm tra database file tồn tại: backend/computerstore.db
  - Chạy seed: npm run seed (nếu có)
