BƯỚC 9: DỪNG VÀ RESTART
================================================================================

Dừng tất cả container:
  docker-compose down

Dừng mà giữ volume:
  docker-compose down -v  (ghi chú: -v sẽ xóa volume)

Restart tất cả container:
  docker-compose restart

Restart một service cụ thể:
  docker-compose restart backend
  docker-compose restart frontend

================================================================================
BƯỚC 10: KIỂM TRA LỖI VÀ TROUBLESHOOTING
================================================================================

1. Nếu container không khởi động:
   - Kiểm tra log: docker-compose logs [service-name]
   - Kiểm tra port có bị chiếm không: netstat -an | findstr :5000
   - Xóa container cũ: docker-compose down --remove-orphans

2. Nếu Build thất bại:
   - Xóa cache: docker-compose build --no-cache
   - Kiểm tra file Dockerfile
   - Kiểm tra package.json

3. Nếu Frontend không kết nối Backend:
   - Kiểm tra REACT_APP_API_URL trong docker-compose.yml
   - Đảm bảo Backend đã khởi động
   - Kiểm tra network: docker network ls

4. Nếu Port bị chiếm:
   - Thay đổi port trong docker-compose.yml
   - Ví dụ: "8000:5000" thay vì "5000:5000"

NOTES & COMMON ISSUES
================================================================================

1. Port Conflicts:
   - Nếu port 5000 hoặc 3000 đã được sử dụng, thay đổi trong docker-compose.yml

2. Slow Build:
   - Lần đầu tiên build sẽ chậm (tải dependencies)
   - Lần kế tiếp sẽ nhanh hơn (Docker cache)

3. Memory Issues:
   - Nếu máy có ít RAM, giảm replicas hoặc resources limits
   - Thêm vào docker-compose.yml:
     deploy:
       resources:
         limits:
           memory: 512M

4. Network Issues:
   - Kiểm tra Docker network hoạt động tốt
   - Kiểm tra firewall không block ports

5. Container Exit:
   - Kiểm tra logs: docker-compose logs [service]
   - Xem exit code: docker-compose ps
   - Debug: docker run -it node:18-alpine /bin/sh
